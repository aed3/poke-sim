Yes, participation 🎉! Contributions from people who are also interested in exploring the programming potential of Pokemon are greatly welcomed and appreciated. Here are the stages of the project and what contributions would be most helpful when:

| Order | Name | Description | Most Useful Contributions |
| --------------- | --------------- | --------------- | --------------- |
| 1 | Architecture Planning | Deciding how to structure the code and adapt from Showdown | Architecture |
| 2 | Vertical Slice 1 | Implementing many of the core functions so a simulation of a specific scenario can run | Architecture, Test Writing |
| 3 | Creating Performance Tests | Creating a systematic way to test how fast the code runs and find areas to improve | Architecture |
| 4 | Vertical Slice 2 | Implementing more of the simulation code so another, more complicated, scenario can run | Test Writing, Implementation |
| 5 | Code Generation Tools | Writing the tools that can read Pokemon data from Showdown and output C++ the equivalent | Test Writing |
| 6 | Event Implementation | Implementing the remaining event handlers and mechanics | Test Writing, Implementation |

Once code implementation contributions begin, I will update this document to include more of my coding guidelines.
